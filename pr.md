# Чек-лист выполнения требований

## Функциональные требования

### Команды бота
- [ ] `/start` - регистрация пользователя.
- [ ] `/help` - вывод списка доступных команд.
- [ ] `/track` - начать отслеживание ссылки (через диалог).
- [ ] `/untrack` - прекратить отслеживание ссылки.
- [ ] `/list` - вывод списка отслеживаемых ссылок.
    - [ ] Вывод специального сообщения, если список пуст.
- [ ] Игнорирование неизвестных команд с уведомлением пользователя.

### Обработка ссылок
- [ ] Проверка дублирования при добавлении ссылки (сообщение "Ссылка уже отслеживается").
- [ ] Модификация `LinkUpdate` в Bot API для расширения данных.

### Интеграция
- [ ] Соответствие всех endpoint'ов OpenAPI-контракту.
- [ ] Реализация базового планировщика уведомлений:
    - [ ] Использование `@Scheduled` (Java), `gocron` (Go), `Scheduler[ZIO]/quartz4s[CE]` (Scala).
    - [ ] Отправка заглушки при обнаружении изменений.
- [ ] Общение между `bot` и `scrapper` по HTTP.

---

## Нефункциональные требования

### Безопасность и конфигурация
- [ ] Токен авторизации хранится в защищенном конфигурационном файле.
- [ ] Типобезопасная конфигурация (примеры для Java/Python/Go/Scala).
- [ ] Запрет на подключение дополнительных библиотек (кроме разрешенных).

### HTTP-клиенты
- [ ] Ручная реализация клиентов для API:
    - [ ] GitHub (без SDK).
    - [ ] StackOverflow (без SDK).
- [ ] Использование декларативных клиентов (по возможности).
- [ ] Проверка обновлений через поля типа `lastUpdated`.

### Диалог и тестирование
- [ ] Реализация машины состояний для `/track` (пошаговый диалог).
    - [ ] Ввод ссылки отдельно от команды.
    - [ ] Опциональные теги и фильтры.
- [ ] Мокирование вызовов к API в тестах (без реальных запросов).

### Логирование
- [ ] Структурное логирование (key-value в логах вместо строковых сообщений).

## Бонус
- [ ] 20 бонусов за автоматическую реализацию меню команд бота (см API), т.е. ваш бот должен регистировать доступные команды через метод setmycommands при запуске
